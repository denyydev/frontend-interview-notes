## Функциональные компоненты vs классовые

В React существует два способа создавать компоненты:
- **функциональные**
- **классовые**

На практике в 2025–2026 году используются почти исключительно **функциональные компоненты**,  
а классовые встречаются в основном в легаси-коде.

---

### Функциональные компоненты

Функциональный компонент — это обычная JavaScript-функция, которая возвращает JSX.

```jsx
function Greeting({ name }) {
  return <h1>Привет, {name}</h1>
}
Особенности:

простой и читаемый синтаксис

меньше шаблонного кода

состояние и побочные эффекты реализуются через hooks

легче тестировать и поддерживать

С появлением hooks функциональные компоненты получили:

состояние (useState)

эффекты (useEffect)

доступ к жизненному циклу

переиспользование логики через custom hooks

Сегодня это основной и рекомендуемый подход.

Классовые компоненты
Классовый компонент — это класс, наследующийся от React.Component.

jsx
Копировать код
class Greeting extends React.Component {
  render() {
    return <h1>Привет, {this.props.name}</h1>
  }
}
Для работы с состоянием и жизненным циклом используются методы класса:

constructor

this.state

setState

componentDidMount

componentDidUpdate

componentWillUnmount

Минусы:

больше шаблонного кода

сложнее читать и поддерживать

логика жизненного цикла размазана по разным методам

сложнее переиспользовать логику

Главное различие
Функциональные	Классовые
Функции	Классы
Hooks	Lifecycle-методы
Меньше кода	Больше шаблона
Проще поддержка	Сложнее поддержка
Стандарт сегодня	Легаси

Что важно на собеседовании
новые компоненты пишутся только функциональными

классы нужно уметь читать, но не обязательно писать

hooks заменили жизненный цикл классов

React официально рекомендует функциональные компоненты

Короткий вывод
Функциональные компоненты — текущий стандарт React.
Классовые компоненты — часть истории и легаси-кода.
